Q51: How does JWT token work?
1. Theory
JWT (JSON Web Token) is a compact, URL-safe token format for securely transmitting information between parties as a JSON object. A JWT consists of three Base64Url-encoded parts separated by dots: Header (algorithm and token type), Payload (claims like user ID, roles, expiration), and Signature (HMAC or RSA signature to verify integrity). When a user logs in, the server generates a JWT signed with a secret key or private key. The client stores this token (typically in localStorage or httpOnly cookie) and sends it in the Authorization: Bearer <token> header with each request. The server validates the signature without querying the database, making JWTs stateless and scalable. However, once issued, JWTs can't be revoked until expiration unless you implement a token blacklist.
2. How it Works in .NET Core
.NET Core uses the System.IdentityModel.Tokens.Jwt package:
  • JwtSecurityTokenHandler - generates and validates JWT tokens
  • AddAuthentication(JwtBearerDefaults.AuthenticationScheme) - configures JWT authentication
  • AddJwtBearer() - configures token validation parameters (issuer, audience, signing key)
  • Claims - stored in the payload (e.g., ClaimTypes.Name, ClaimTypes.Role)
  • TokenValidationParameters - defines validation rules (lifetime, issuer, audience, signature)
  • [Authorize] attribute - protects endpoints, requires valid JWT
3. Practical Example
In a multi-tenant SaaS platform serving 50K users, we implemented JWT-based authentication with refresh tokens. JWTs had a 15-minute expiration, and refresh tokens lasted 7 days stored in httpOnly cookies. This reduced database lookups for session validation from 200K/min to near zero, improving API response times by 40ms on average. 
// Program.cs - JWT Configuration

using System.IdentityModel.Tokens.Jwt;
using System.Security.Claims;
using Microsoft.IdentityModel.Tokens;

public interface IJwtService
{
    string GenerateToken(User user);
    ClaimsPrincipal? ValidateToken(string token);
}

public class JwtService : IJwtService
{
    private readonly IConfiguration _configuration;

    public JwtService(IConfiguration configuration)
    {
        _configuration = configuration;
    }

    public string GenerateToken(User user)
    {
        var securityKey = new SymmetricSecurityKey(
            Encoding.UTF8.GetBytes(_configuration["Jwt:SecretKey"]));
        
        var credentials = new SigningCredentials(securityKey, SecurityAlgorithms.HmacSha256);

        // Define claims (payload data)
        var claims = new[]
        {
            new Claim(JwtRegisteredClaimNames.Sub, user.Id.ToString()),
            new Claim(JwtRegisteredClaimNames.Email, user.Email),
            new Claim(JwtRegisteredClaimNames.Jti, Guid.NewGuid().ToString()),
            new Claim(ClaimTypes.Name, user.Username),
            new Claim(ClaimTypes.Role, user.Role),
            new Claim("tenant_id", user.TenantId.ToString()) // Custom claim
        };

        var token = new JwtSecurityToken(
            issuer: _configuration["Jwt:Issuer"],
            audience: _configuration["Jwt:Audience"],
            claims: claims,
            expires: DateTime.UtcNow.AddMinutes(15), // Short-lived token
            signingCredentials: credentials
        );

        return new JwtSecurityTokenHandler().WriteToken(token);
    }

    public ClaimsPrincipal? ValidateToken(string token)
    {
        var tokenHandler = new JwtSecurityTokenHandler();
        var key = Encoding.UTF8.GetBytes(_configuration["Jwt:SecretKey"]);

        try
        {
            var principal = tokenHandler.ValidateToken(token, new TokenValidationParameters
            {
                ValidateIssuerSigningKey = true,
                IssuerSigningKey = new SymmetricSecurityKey(key),
                ValidateIssuer = true,
                ValidIssuer = _configuration["Jwt:Issuer"],
                ValidateAudience = true,
                ValidAudience = _configuration["Jwt:Audience"],
                ValidateLifetime = true,
                ClockSkew = TimeSpan.Zero
            }, out SecurityToken validatedToken);

            return principal;
        }
        catch
        {
            return null; // Invalid token
        }
    }
}

Q52: What happens when JWT is expired and what will be the result?
1. Theory
When a JWT expires, the server rejects the token during validation and returns a 401 Unauthorized response. The expiration is determined by the exp (expiration time) claim in the JWT payload, which is a Unix timestamp. The server compares the current time with the exp claim—if current time is past expiration, the token is invalid. This triggers a SecurityTokenExpiredException in .NET. Clients should catch 401 responses and attempt to refresh the token using a refresh token. Without a refresh mechanism, the user must log in again. The ClockSkew setting (default 5 minutes) provides a buffer for clock differences between servers.
2. How it Works in .NET Core
Expired token handling in .NET:
  • TokenValidationParameters.ValidateLifetime = true - enables expiration checking
  • SecurityTokenExpiredException - thrown when token is expired
  • 401 Unauthorized response - automatic response from middleware
  • ClockSkew - allows time drift tolerance (default 5 minutes)
  • Refresh tokens - long-lived tokens to obtain new access tokens
  • JwtBearerEvents.OnAuthenticationFailed - hook to customize expired token handling
3. Practical Example
In a banking API, we implemented automatic token refresh to avoid disrupting user sessions during transactions. When a JWT expired mid-transaction, the client detected the 401 response, used the refresh token to get a new JWT, and retried the request automatically. This reduced failed transactions by 12% and improved user satisfaction scores. 
// Program.cs - Handle Expired Tokens
builder.Services.AddAuthentication(JwtBearerDefaults.AuthenticationScheme)
    .AddJwtBearer(options =>
    {
        options.TokenValidationParameters = new TokenValidationParameters
        {
            ValidateLifetime = true, // Check expiration
            ClockSkew = TimeSpan.Zero, // No tolerance (strict expiration)
            // ... other validation parameters
        };

        options.Events = new JwtBearerEvents
        {
            OnAuthenticationFailed = context =>
            {
                if (context.Exception is SecurityTokenExpiredException)
                {
                    // Add custom header to help client handle expiration
                    context.Response.Headers.Add("Token-Expired", "true");
                    context.Response.Headers.Add("Access-Control-Expose-Headers", "Token-Expired");
                }
                return Task.CompletedTask;
            },
            OnChallenge = context =>
            {
                // Customize 401 response for expired tokens
                context.HandleResponse();
                context.Response.StatusCode = StatusCodes.Status401Unauthorized;
                context.Response.ContentType = "application/json";

                var result = JsonSerializer.Serialize(new
                {
                    error = "token_expired",
                    message = "Your session has expired. Please refresh your token."
                });

                return context.Response.WriteAsync(result);
            }
        };
    });

// Refresh Token Implementation
public record RefreshToken
{
    public string Token { get; init; }
    public string UserId { get; init; }
    public DateTime ExpiresAt { get; init; }
    public bool IsRevoked { get; set; }
}

public class TokenService
{
    private readonly IJwtService _jwtService;
    private readonly AppDbContext _context;

    public TokenService(IJwtService jwtService, AppDbContext context)
    {
        _jwtService = jwtService;
        _context = context;
    }

    public async Task<TokenResponse> GenerateTokenPairAsync(User user)
    {
        // Generate short-lived access token (15 minutes)
        var accessToken = _jwtService.GenerateToken(user);

        // Generate long-lived refresh token (7 days)
        var refreshToken = new RefreshToken
        {
            Token = Convert.ToBase64String(RandomNumberGenerator.GetBytes(64)),
            UserId = user.Id.ToString(),
            ExpiresAt = DateTime.UtcNow.AddDays(7),
            IsRevoked = false
        };

        _context.RefreshTokens.Add(refreshToken);
        await _context.SaveChangesAsync();

        return new TokenResponse
        {
            AccessToken = accessToken,
            RefreshToken = refreshToken.Token,
            ExpiresIn = 900 // 15 minutes
        };
    }

    public async Task<TokenResponse?> RefreshAccessTokenAsync(string refreshTokenValue)
    {
        var refreshToken = await _context.RefreshTokens
            .FirstOrDefaultAsync(rt => rt.Token == refreshTokenValue);

        if (refreshToken == null || 
            refreshToken.IsRevoked || 
            refreshToken.ExpiresAt < DateTime.UtcNow)
        {
            return null; // Invalid or expired refresh token
        }

        var user = await _context.Users.FindAsync(refreshToken.UserId);
        if (user == null) return null;

        // Generate new token pair
        var newTokenPair = await GenerateTokenPairAsync(user);

        // Revoke old refresh token (one-time use)
        refreshToken.IsRevoked = true;
        await _context.SaveChangesAsync();

        return newTokenPair;
    }
}

// Logout - Revoke refresh token
app.MapPost("/api/auth/logout", [Authorize] async (
    HttpContext context,
    AppDbContext db) =>
{
    var userId = context.User.FindFirst(ClaimTypes.NameIdentifier)?.Value;
    
    // Revoke all refresh tokens for this user
    var tokens = await db.RefreshTokens
        .Where(rt => rt.UserId == userId && !rt.IsRevoked)
        .ToListAsync();

    tokens.ForEach(t => t.IsRevoked = true);
    await db.SaveChangesAsync();

    return Results.Ok(new { message = "Logged out successfully" });
});

axios.interceptors.response.use(
    response => response,
    async error => {
        const originalRequest = error.config;

        // Check if error is 401 and token expired
        if (error.response?.status === 401 && 
            error.response?.headers['token-expired'] === 'true' &&
            !originalRequest._retry) {
            
            originalRequest._retry = true;

            try {
                // Attempt to refresh token
                const refreshToken = localStorage.getItem('refresh_token');
                const response = await axios.post('/api/auth/refresh', { refreshToken });
                
                // Store new tokens
                localStorage.setItem('access_token', response.data.accessToken);
                localStorage.setItem('refresh_token', response.data.refreshToken);

                // Retry original request with new token
                originalRequest.headers.Authorization = `Bearer ${response.data.accessToken}`;
                return axios(originalRequest);
            } catch (refreshError) {
                // Refresh failed - redirect to login
                window.location.href = '/login';
                return Promise.reject(refreshError);
            }
        }

        return Promise.reject(error);
    }

Q53: What is the difference between authentication and authorization in .NET Core?
1. Theory
Authentication answers "Who are you?"—it verifies the user's identity through credentials (username/password, JWT, OAuth, certificates). Authorization answers "What can you do?"—it determines if the authenticated user has permission to access a resource based on roles, policies, or claims. Authentication happens first in the request pipeline, followed by authorization. For example, a user might authenticate successfully with a JWT (proving identity) but fail authorization when trying to access an admin-only endpoint (lacking permission). In .NET, authentication is handled by UseAuthentication() middleware, and authorization by UseAuthorization() middleware.
2. How it Works in .NET Core
Authentication:
  • AddAuthentication() - registers authentication services
  • UseAuthentication() - middleware that validates credentials
  • Schemes: JwtBearer, Cookies, OpenIdConnect, Windows
  • Sets HttpContext.User with ClaimsPrincipal if valid
Authorization:
  • AddAuthorization() - registers authorization services
  • UseAuthorization() - middleware that checks permissions
  • [Authorize] - requires authentication
  • [Authorize(Roles = "Admin")] - role-based authorization
  • [Authorize(Policy = "MinimumAge")] - policy-based authorization
  • Custom requirements via IAuthorizationRequirement
3. Practical Example
In an e-learning platform with 100K users, we used JWT for authentication (proving student identity) and policy-based authorization for content access (checking course enrollment, subscription level, and age restrictions). For example, a student could authenticate successfully but be denied access to premium courses if they didn't have an active subscription. 
// Program.cs - Authentication and Authorization Setup

            ValidateAudience = true,
            ValidateLifetime = true,
            ValidateIssuerSigningKey = true,
            ValidIssuer = builder.Configuration["Jwt:Issuer"],
            ValidAudience = builder.Configuration["Jwt:Audience"],
            IssuerSigningKey = new SymmetricSecurityKey(
                Encoding.UTF8.GetBytes(builder.Configuration["Jwt:SecretKey"]))

    });

// 2. AUTHORIZATION - What can you do?
builder.Services.AddAuthorization(options =>
{
    // Role-based policy
    options.AddPolicy("AdminOnly", policy => 
        policy.RequireRole("Admin"));

    // Claims-based policy
    options.AddPolicy("PremiumSubscriber", policy => 
        policy.RequireClaim("subscription_level", "Premium", "Enterprise"));

    options.AddPolicy("MinimumAge18", policy =>
        policy.Requirements.Add(new MinimumAgeRequirement(18)));

    // Combined requirements
    options.AddPolicy("CourseInstructor", policy =>
        policy.RequireRole("Instructor")
              .RequireClaim("verified", "true"));
});

// Custom Authorization Requirement
public class MinimumAgeRequirement : IAuthorizationRequirement
{
    public int MinimumAge { get; }

    public MinimumAgeRequirement(int minimumAge)
    {
        MinimumAge = minimumAge;
    }
}

public class MinimumAgeHandler : AuthorizationHandler<MinimumAgeRequirement>
{
    protected override Task HandleRequirementAsync(
        AuthorizationHandlerContext context,
        MinimumAgeRequirement requirement)
    {
        var dateOfBirthClaim = context.User.FindFirst(c => c.Type == "date_of_birth");

        if (dateOfBirthClaim == null)
        {
            return Task.CompletedTask; // Fail - no DOB claim
        }

        if (DateTime.TryParse(dateOfBirthClaim.Value, out var dateOfBirth))
        {
            var age = DateTime.Today.Year - dateOfBirth.Year;
            if (dateOfBirth.Date > DateTime.Today.AddYears(-age)) age--;

            if (age >= requirement.MinimumAge)
            {
                context.Succeed(requirement); // Pass - user meets age requirement
            }
        }

        return Task.CompletedTask;
    }
}

// Resource-based Authorization (checking specific resource ownership)
public class CourseAuthorizationHandler : 
    AuthorizationHandler<SameAuthorRequirement, Course>
{
    protected override Task HandleRequirementAsync(
        AuthorizationHandlerContext context,
        SameAuthorRequirement requirement,
        Course resource)
    {
        var userId = context.User.FindFirst(ClaimTypes.NameIdentifier)?.Value;

        if (userId == resource.AuthorId)
        {
            context.Succeed(requirement); // User owns this course
        }

        return Task.CompletedTask;
    }
}

app.MapPut("/api/courses/{id}", [Authorize] async (
    int id,
    UpdateCourseRequest request,
    ICourseService courseService,
    IAuthorizationService authService,
    HttpContext context) =>
{
    var course = await courseService.GetByIdAsync(id);
    if (course == null) return Results.NotFound();

    // Check if user is authorized to modify THIS specific course
    var authResult = await authService.AuthorizeAsync(
        context.User,

        new SameAuthorRequirement());

    if (!authResult.Succeeded)
        return Results.Forbid(); // 403 Forbidden

    await courseService.UpdateAsync(id, request);
    return Results.NoContent();
});

4. Key Takeaway
Authentication = Identity verification ("Who are you?"). Authorization = Permission checking ("What can you access?"). Authentication happens first via UseAuthentication(), followed by authorization via UseAuthorization(). Use [Authorize] for authentication, [Authorize(Roles)] for role-based authorization, and [Authorize(Policy)] for complex business rules. Always implement both—authenticated users should still be denied access to resources they don't have permission to access.
